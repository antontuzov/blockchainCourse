# Week 7: Permissioned Blockchains & Hyperledger

## Topics
- Permissioned vs permissionless blockchains
- Hyperledger Fabric architecture
- Channels and private data
- Identity management
- Certificate authorities

## Learning Objectives
By the end of this week, students will be able to:
1. Understand the differences between permissioned and permissionless blockchains
2. Implement a permissioned blockchain with role-based access control
3. Create and manage channels for private data
4. Implement identity management systems
5. Work with certificate authorities for authentication

## Theory Session (2 hours)
### 1. Permissioned vs Permissionless Blockchains (30 min)
- Definitions and characteristics
- Use cases and applications
- Security considerations

### 2. Hyperledger Fabric Architecture (45 min)
- Modular architecture
- Components overview
- Network topology

### 3. Identity Management (45 min)
- Digital identities
- Certificate authorities
- Role-based access control

## Implementation Session (3 hours)
### 1. Permissioned Blockchain Implementation (90 min)
- Membership services
- Access control policies
- Role management

### 2. Channel Management (60 min)
- Channel creation
- Private data collections
- Channel policies

### 3. Identity Management System (30 min)
- Identity creation and management
- Certificate authority integration
- Authentication mechanisms

## Project Work (2 hours)
### Hands-on Projects:
1. Membership service implementation
2. Channel management system
3. Access control policies
4. Private data collections

## Assessment
- Code review of implemented permissioned blockchain
- Channel management testing
- Quiz on identity management
